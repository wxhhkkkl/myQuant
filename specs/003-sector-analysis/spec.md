# Feature Specification: 板块分析模块

**Feature Branch**: `003-sector-analysis`
**Created**: 2026-05-14
**Status**: Draft
**Input**: User description: "新建一个板块分析模块，可以让我能挑选出近一年内板块轮动的状态，看到哪些板块总体估值偏低，每个板块在一年内有了几次行情，当前最热的板块是什么，完后每个板块可以查看板块内的股票情况，板块的趋势为板块内的所有股票的价格的集合"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 板块概览看板 (Priority: P1)

投资者打开板块分析页面，一眼看到当前市场的板块全景：哪些板块估值偏低、哪些板块近期热度最高、各板块在近一年内经历过几轮行情。这个看板帮助投资者快速判断当前市场风格（价值/成长、进攻/防御），为选股方向提供宏观依据。

**Why this priority**: 板块概览是模块的入口和核心价值所在。没有概览，用户无法进行任何板块分析。所有后续的轮动分析、板块详情都依赖于此。

**Independent Test**: 打开板块分析页面，即可看到板块列表及每个板块的估值分位、近一年行情次数、当前热度排名。无需任何交互即可获得有效信息。

**Acceptance Scenarios**:

1. **Given** 系统已有至少一年的板块K线数据和成分股估值数据，**When** 用户打开板块分析页面，**Then** 页面展示所有申万一级行业板块列表，每行包含板块名称、当前估值分位（高/中/低）、近一年行情次数、近一周涨跌幅、当前热度排名
2. **Given** 板块概览已加载，**When** 用户按"热度排名"排序，**Then** 板块列表按热度降序重新排列，最热板块置顶
3. **Given** 板块概览已加载，**When** 某板块没有任何K线数据（如新上市板块），**Then** 该板块显示"数据不足"而非报错

---

### User Story 2 - 板块轮动时间线 (Priority: P2)

投资者选择一个时间范围（默认近一年），系统展示该时间段内的板块轮动热力图或时间线：哪些板块在哪些时段主导市场、领涨板块如何切换。这让投资者理解资金在不同板块间的流动节奏，识别轮动规律。

**Why this priority**: 轮动分析是板块分析区别于个股分析的核心差异化功能。但它在概览之后，因为用户需要先看到当前状态，再深入分析变化过程。

**Independent Test**: 在板块分析页面选择"近一年"时间范围，即可看到轮动图，展示每月领涨板块的切换情况。

**Acceptance Scenarios**:

1. **Given** 板块概览已展示，**When** 用户切换到"轮动分析"视图并选择"近6个月"，**Then** 系统默认按月展示这6个月内每月的领涨板块（涨幅前3名），用户可切换为周度视图
2. **Given** 轮动视图已加载，**When** 用户点击时间线上的某个时间点，**Then** 展开该时点的板块涨跌排名详情
3. **Given** 板块轮动分析需要计算板块区间涨跌幅，**When** 某板块在选定区间内没有完整数据，**Then** 该板块在对应时间段标注"数据不完整"

---

### User Story 3 - 板块详情与成分股 (Priority: P3)

投资者点击某个板块，进入板块详情页。查看板块的趋势图（板块内所有股票价格等权集合）、成分股列表及其各自的涨跌幅和估值。用户可以在这个页面评估该板块是否值得配置，以及板块内哪些个股最有潜力。

**Why this priority**: 板块详情是分析的终点——用户从概览发现机会，通过轮动验证节奏，最后深入具体板块做投资决策。没有前两步，直接看详情缺乏上下文。

**Independent Test**: 在板块概览点击任一板块名称，进入详情页，看到板块趋势K线图、成分股表格。

**Acceptance Scenarios**:

1. **Given** 用户在板块概览页，**When** 点击"食品饮料"板块，**Then** 跳转到板块详情页，展示该板块的趋势K线图（近一年，可切换周期）、成分股列表（含代码、名称、最新价、涨跌幅、市盈率、市值）
2. **Given** 板块趋势图已展示，**When** 用户切换K线周期（日/周/月），**Then** 图表按选中周期重新渲染
3. **Given** 成分股列表已展示，**When** 用户点击某只成分股，**Then** 跳转到该股票的个股详情页

---

### Edge Cases

- 某板块成分股为空（如行业分类变更导致）时，板块卡片显示"暂无成分股"且趋势图不可用
- 板块估值数据缺失时，估值分位显示"--"而非错误值0
- 用户选择的时间范围内市场整体处于单边下跌，所有板块热度都低——系统仍正常显示，不因绝对值误导
- 同一只股票属于多个板块时（如同时属于一级行业和二级行业），在各自板块内独立展示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 基于申万一级行业分类展示所有板块的概览列表
- **FR-002**: 系统 MUST 计算并展示每个板块的当前估值分位（将所有板块按PE中位数排名，前1/3为"高估"，中1/3为"适中"，后1/3为"低估"）
- **FR-003**: 系统 MUST 识别并统计每个板块在近一年内的"行情次数"（连续涨幅≥10%且持续≥5个交易日的上涨区间计为一次行情）
- **FR-004**: 系统 MUST 计算并展示板块"当前热度"排名（综合近一周涨跌幅、成交量变化、成分股上涨比例）
- **FR-005**: 系统 MUST 提供板块轮动时间线视图，展示用户选定时间范围内各时段领涨板块（涨幅前3名）及切换轨迹，支持月度/周度切换，默认月度
- **FR-006**: 系统 MUST 在用户手动触发后预计算板块趋势指数（板块内所有成分股价格的等权集合）并持久化存储，页面查询直接读取预计算结果并以K线图展示。板块分析页面 MUST 显示上一次数据更新时间，并提供"更新板块信息"按钮供用户手动刷新
- **FR-007**: 系统 MUST 支持用户从板块概览点击进入板块详情页，查看趋势图和完整成分股列表
- **FR-008**: 系统 MUST 允许用户按估值分位、热度排名、涨跌幅对板块列表排序
- **FR-009**: 系统 MUST 允许用户选择分析的时间范围（1个月 / 3个月 / 6个月 / 1年）
- **FR-010**: 板块趋势K线 MUST 支持日线、周线、月线切换

### Key Entities

- **板块 (Sector)**: 行业分类单元，包含板块名称、行业代码、成分股数量。对应申万一级行业分类（如食品饮料、医药生物、电子等）
- **板块行情 (SectorMovement)**: 板块在一段时间内的涨跌表现，包含起止日期、区间涨跌幅、是否为领涨板块
- **板块趋势 (SectorTrend)**: 板块内所有成分股价格的等权集合形成的合成K线（开/高/低/收），用于绘制板块走势图
- **板块热度 (SectorHeat)**: 综合评分指标，由近一周涨跌幅、成交量环比变化、成分股上涨比例三者加权计算

## Clarifications

### Session 2026-05-14

- Q: 板块轮动时间线的"时间粒度"？ → A: 用户可选（月/周切换，默认月度）
- Q: 板块趋势指数存储策略？ → A: 预计算存储（定时任务预计算板块K线并持久化到新表）
- Q: 板块分析数据刷新频率？ → A: 手动触发，页面上提供"更新板块信息"按钮，并显示上一次更新时间
- Q: 板块估值分位的"高/中/低"三档如何划分？ → A: 三等分法——所有板块按PE中位数排名，前1/3=高估，中1/3=适中，后1/3=低估

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户打开板块分析页面后，3秒内看到完整的板块概览列表（含估值分位、行情次数、热度排名）
- **SC-002**: 板块轮动时间线能清晰展示近一年内至少10个时间节点的领涨板块切换
- **SC-003**: 用户从板块概览点击进入板块详情，2秒内看到趋势图与成分股列表
- **SC-004**: 板块热度排名每周更新后，排名变化与市场实际热点板块一致（人工抽查5个热门板块验证）
- **SC-005**: 用户能在板块分析模块中完成"概览 → 轮动 → 详情 → 个股"的完整分析流程，无需离开模块

## Assumptions

- 板块分类沿用项目已有的申万行业分类数据（存储在 `stocks.industry` 字段中）
- 板块估值分位基于成分股PE中位数计算，而非板块整体市值加权PE
- "行情次数"的阈值默认设定为：连续涨幅≥10%且持续≥5个交易日
- 板块趋势指数默认采用等权方式计算（各成分股权重相同）
- 热度排名公式：近一周涨跌幅×0.4 + 成交量环比变化×0.3 + 成分股上涨比例×0.3
- 数据依赖项目已有的 daily_kline（DuckDB）、stocks 表（SQLite）、stock_fundamentals 表（SQLite）
- 板块分析页面需在现有导航栏中新增入口，与"个股列表""量化模型""回测""交易""账户"并列
