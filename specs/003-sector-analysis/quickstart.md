# Quickstart: 板块分析模块

**Feature**: 003-sector-analysis | **Date**: 2026-05-14

## 前置条件

1. 数据库已有股票基础信息和行业分类（`stocks.industry` 字段已填充）
2. DuckDB `daily_kline` 至少有一年的日K线数据
3. SQLite `stock_fundamentals` 有最新的 PE 数据
4. Python 3.11+ 环境，FastAPI 应用可正常运行

## 初次使用步骤

### 1. 启动应用

```bash
cd backend
python -m backend.src.main
```

### 2. 打开板块分析页面

访问 `http://localhost:8000/sector-analysis`

首次访问时板块列表为空，需点击 **"更新板块信息"** 按钮。

### 3. 触发板块数据计算

点击页面上的 **"更新板块信息"** 按钮，系统将在后台计算：
- 31 个申万一级行业的板块趋势K线（等权合成日线）
- 各板块估值分位（三等分法基于 PE 中位数排名）
- 近一年行情次数（≥10% 涨幅且 ≥5 个交易日）
- 热度排名（涨跌幅 ×0.4 + 成交量环比 ×0.3 + 上涨比例 ×0.3）

预计耗时约 10-30 秒，进度条实时显示状态。完成后页面自动刷新。

### 4. 使用三大功能

- **概览 Tab**: 查看板块列表，支持按热度/估值/涨跌幅排序
- **轮动 Tab**: 按月/周查看领涨板块切换，点击时间节点看详情
- **详情 Tab**: 点击板块名称进入详情页，查看趋势K线图和成分股

## 数据更新

板块数据不会自动更新。每次需要最新数据时，手动点击"更新板块信息"按钮。

注意：更新时需要访问 DuckDB 文件，如果 FastAPI 应用正在运行且占用 DuckDB，计算脚本会因文件锁而失败。此时需先停止应用 → 运行计算 → 重启应用。（未来可考虑通过 API 触发后台线程更新，避免锁冲突。）

## 手动运行计算脚本

也可以从命令行直接运行：

```bash
python -m backend.src.scripts.compute_sectors
```

此方式需先停止 FastAPI 应用以释放 DuckDB 锁。
