# Feature Specification: 个股列表与详情模块

**Feature Branch**: `002-stock-list`
**Created**: 2026-05-11
**Status**: Draft
**Input**: User description: "增加一个模块叫个股列表，其中有自选股，完后有更新所有个股基础信息的功能；点击每个股票有个人自己的详情，可以看到K线图和行情信息"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 浏览个股列表 (Priority: P1)

用户打开个股列表模块，可以看到所有A股股票的基本信息列表，包括股票代码、名称、最新价、涨跌幅等关键字段，并支持分页浏览和简单搜索筛选。

**Why this priority**: 个股列表是整个模块的入口和核心视图，没有列表就无法访问任何单只股票的详情或自选股功能。

**Independent Test**: 打开个股列表页面，能看到股票数据以分页列表形式展示，包含代码、名称、价格、涨跌幅等基本信息。

**Acceptance Scenarios**:

1. **Given** 系统中已存储股票基础信息，**When** 用户进入个股列表页面，**Then** 默认显示第一页股票列表，按代码排序，每页显示合理数量，包含股票代码、名称、最新价、涨跌幅字段。
2. **Given** 用户在个股列表页面，**When** 用户在搜索框输入股票代码或名称关键字，**Then** 列表实时过滤显示匹配的股票。
3. **Given** 系统中股票数量超过一页，**When** 用户点击下一页或页码，**Then** 显示对应页的股票数据。
4. **Given** 用户在个股列表页面，**When** 用户点击"涨跌幅"或"最新价"列头，**Then** 列表按该字段重新排序，再次点击切换升序/降序。

---

### User Story 2 - 个股详情页 (Priority: P1)

用户从个股列表中点击某只股票，进入该股票的详情页面，查看K线图（可切换不同周期）和实时行情信息。

**Why this priority**: 个股详情是用户做出投资决策的核心页面，K线图和行情信息是最基础、最高频的需求。

**Independent Test**: 点击任意一只股票，能进入其详情页，看到K线图和行情数据面板。

**Acceptance Scenarios**:

1. **Given** 用户在个股列表页面，**When** 点击某只股票的代码或名称，**Then** 跳转至该股票的详情页，默认显示近一年（约250个交易日）日K线图和当前行情信息。
2. **Given** 用户在个股详情页，**When** 切换K线周期（日K/周K/月K），**Then** 图表刷新显示对应周期的K线数据。
3. **Given** 用户在个股详情页，**When** 页面加载完成，**Then** 显示该股票的最新价、涨跌幅、涨跌额、成交量、成交额、最高价、最低价、开盘价、昨收价等行情字段，并提供手动刷新按钮。
4. **Given** 用户已在个股详情页停留一段时间，**When** 点击刷新按钮，**Then** 行情数据更新为最新快照。
5. **Given** 某只股票当天无交易数据，**When** 用户进入其详情页，**Then** 显示最近一个交易日的K线数据和行情快照，并提示"当前为非交易日"。

---

### User Story 3 - 自选股管理 (Priority: P2)

用户可以将关注的股票添加到自选股列表，在个股列表中通过自选股筛选快速查看，也可以将股票从自选股中移除。

**Why this priority**: 自选股是用户个性化投资管理的核心功能，帮助用户快速定位关注的标的，但依赖于P1的列表基础能力。

**Independent Test**: 在个股列表中对某只股票执行添加自选操作，切换到自选股筛选后仅显示已添加的股票；执行移除操作后该股票不再出现在自选股筛选中。

**Acceptance Scenarios**:

1. **Given** 用户在个股列表页面，**When** 点击某只股票旁边的"加自选"按钮，**Then** 该股票被加入自选股列表，按钮状态变更为"已自选"。
2. **Given** 用户已将若干股票加入自选股，**When** 在个股列表中选择"仅看自选"筛选条件，**Then** 列表仅显示已加入自选股的股票。
3. **Given** 用户在个股列表页面且某股票已在自选股中，**When** 点击"取消自选"按钮，**Then** 该股票从自选股中移除，按钮恢复为"加自选"状态。
4. **Given** 用户的自选股列表为空，**When** 选择"仅看自选"筛选，**Then** 显示空状态提示"暂无自选股，请先添加"。

---

### User Story 4 - 更新个股基础信息 (Priority: P2)

用户可以触发批量更新所有个股的基础信息，系统从数据源获取最新的股票基础数据（如股票代码、名称、上市日期、行业分类等），并在更新完成后给出结果反馈。

**Why this priority**: 基础信息是列表和详情的数据基础，但更新是低频操作（股票基础信息变动不频繁），因此优先级略低于详情页和自选股核心交互。

**Independent Test**: 点击"更新基础信息"按钮后，系统开始执行更新，完成后显示更新结果（成功/失败数量）。

**Acceptance Scenarios**:

1. **Given** 用户在个股列表页面，**When** 点击"更新个股基础信息"按钮，**Then** 系统开始批量获取最新股票基础数据，页面显示更新进度或加载状态。
2. **Given** 批量更新正在进行中，**When** 更新完成，**Then** 页面显示本次更新的结果摘要（更新总数、成功数、失败数）。
3. **Given** 批量更新过程中发生部分失败，**When** 更新结束，**Then** 成功的股票数据已更新，失败的股票保留原有数据，并列出失败列表。

---

### Edge Cases

- 股票退市或代码变更时，列表中应标记异常状态而非直接隐藏或报错。
- 非交易时段进入个股详情页，行情数据显示最近一个交易日的收盘快照，K线图可正常展示。
- 用户快速连续点击"加自选/取消自选"时，最终状态应与最后一次操作一致，不出现重复或冲突。
- 更新基础信息时如果数据源不可用，应给出明确错误提示而不影响已有的列表数据展示。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供个股列表页面，以分页表格形式展示所有A股股票，默认按股票代码排序，支持按涨跌幅、最新价排序（升序/降序切换）。
- **FR-002**: 个股列表 MUST 显示每只股票的代码、名称、最新价、涨跌幅。
- **FR-003**: 系统 MUST 支持按股票代码或名称关键字搜索筛选个股列表。
- **FR-004**: 系统 MUST 提供个股详情页面，展示该股票的K线图和实时行情数据。
- **FR-005**: K线图 MUST 支持日K、周K、月K三种周期的切换。
- **FR-006**: 行情数据 MUST 包含最新价、涨跌幅、涨跌额、成交量、成交额、最高价、最低价、开盘价、昨收价。
- **FR-007**: 用户 MUST 能够将股票加入自选股列表，也能从自选股中移除。
- **FR-008**: 个股列表 MUST 支持"仅看自选"的筛选模式，只显示用户已加入自选的股票。
- **FR-009**: 系统 MUST 提供"更新个股基础信息"功能，批量从数据源获取并更新所有股票的基础信息（代码、名称、行业分类等）。
- **FR-010**: 批量更新完成后 MUST 向用户展示更新结果摘要（总数、成功数、失败数）。
- **FR-011**: 自选股列表 MUST 持久化存储，用户重新打开应用后自选股数据不丢失。

### Key Entities

- **Stock（个股）**: 股票基础信息，包含股票代码、名称、行业分类、上市日期、交易状态等。
- **Watchlist（自选股）**: 用户与股票的收藏关系，关联用户和股票代码，包含添加时间。
- **MarketQuote（行情快照）**: 某只股票在某个时刻的实时行情数据，包含最新价、涨跌幅、成交量、成交额、最高/最低/开盘/昨收价。
- **KLine（K线数据）**: 股票的历史K线数据，包含日期、周期、开盘价、收盘价、最高价、最低价、成交量。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 个股列表页面在3秒内完成首屏加载和数据展示。
- **SC-002**: 个股详情页在3秒内完成K线图和行情数据的加载渲染。
- **SC-003**: 用户能在2次点击内将一只股票加入自选股（一次点击"加自选"，无需其他步骤）。
- **SC-004**: 用户能在3次点击内从个股列表导航到任意一只股票的详情页（搜索→点击→进入）。
- **SC-005**: K线周期切换在2秒内完成图表刷新。
- **SC-006**: 批量更新5000只股票的基础信息在60秒内完成。
- **SC-007**: 自选股数据在应用重启后完整保留，无数据丢失。

## Assumptions

- 目标用户为单一个人投资者，无需多用户权限隔离。
- 股票基础信息数据源为 QMT xtdata（已在现有项目中集成），行情和K线数据同样来自 xtdata。
- 前端技术栈延续现有项目的 HTMX + Alpine.js + Tailwind CSS 方案，K线图使用轻量级图表库。
- 仅覆盖A股沪深两市，不包含港股、美股等其他市场。
- K线图交互为查看模式（缩放、切换周期），不包含画线工具或技术指标叠加（留待后续迭代）。

## Clarifications

### Session 2026-05-11

- Q: 个股详情页的K线图默认应展示多长时间范围的历史数据？ → A: 近一年（约250个交易日）
- Q: 详情页行情数据是否需要自动刷新？ → A: 仅手动刷新（用户点击刷新按钮获取最新数据）
- Q: 个股列表是否支持按其他字段排序？ → A: 支持按涨跌幅、最新价排序（升序/降序切换）
