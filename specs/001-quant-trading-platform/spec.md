# Feature Specification: 量化交易平台 (Quant Trading Platform)

**Feature Branch**: `001-quant-trading-platform`
**Created**: 2026-05-03
**Status**: Draft
**Input**: User description: "想做一个自己的量化工具，从AI选股，获取股票的基本面信息与财务报表，收集分析该股票相关的舆情信息以及对应的版块信息，会持续增加多种量化的模型以及对应得回测，第一个量化模型是双均线模型（5日均线和20日均线相交判断买入或卖出），可以执行在线量化交易，实时观察交易情况和盈利情况，可以查看正在使用得交易账户得持仓、收益等情况。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 股票研究与AI选股 (Priority: P1)

用户希望从海量股票中快速筛选出有投资价值的标的。用户可以通过AI选股功能获得智能推荐的股票列表，点击任意股票可查看其基本信息（公司概况、行业板块）、财务数据（营收、利润、资产负债表等关键指标）、以及相关的舆情资讯（新闻、公告、社交媒体讨论）。系统整合多维度信息，帮助用户做出投资决策。

**Why this priority**: 股票数据是整个量化平台的基础。没有股票信息和筛选能力，后续的量化模型和交易都无从谈起。此功能独立交付即可为用户提供研究价值。

**Independent Test**: 用户输入/选择一只股票代码，即可查看其完整的基本面、财务和舆情信息，无需后续功能模块即可验证数据完整性和展示效果。

**Acceptance Scenarios**:

1. **Given** 用户打开AI选股页面，**When** 用户点击"AI选股"，**Then** 系统展示推荐的股票列表，每只股票包含名称、代码、推荐理由
2. **Given** 用户在股票列表中，**When** 用户点击某只股票，**Then** 系统展示该股票的基本面信息（公司全称、行业、市值、PE/PB等）
3. **Given** 用户在某只股票的详情页，**When** 用户切换到"财务数据"标签，**Then** 系统展示近三年的营收、净利润、ROE、资产负债率等关键财务指标
4. **Given** 用户在某只股票的详情页，**When** 用户切换到"舆情分析"标签，**Then** 系统展示该股票相关的新闻、公告及情感倾向分析（正面/负面/中性）
5. **Given** 用户在某只股票的详情页，**When** 用户查看"板块信息"，**Then** 系统展示该股票所属行业板块及板块整体走势

---

### User Story 2 - 双均线量化模型与回测 (Priority: P2)

用户希望使用双均线模型（5日均线和20日均线）对股票进行回测，验证该策略的历史表现。用户可以设置回测参数（时间区间、初始资金），系统根据5日均线上穿20日均线（金叉买入）、5日均线下穿20日均线（死叉卖出）的规则运行模拟交易，并展示回测结果，包括总收益率、年化收益、最大回撤、交易次数、胜率等关键指标。

**Why this priority**: 量化模型是平台的核心差异化能力。双均线作为第一个模型，同时也是模型框架的验证案例。回测结果让用户在实盘前验证策略有效性。

**Independent Test**: 选择一只股票，设定回测时间范围，运行双均线回测，即可查看完整的回测报告和交易明细，不依赖实盘交易功能。

**Acceptance Scenarios**:

1. **Given** 用户在模型页面选择了双均线模型，**When** 用户选择股票并设定回测时间范围（如近一年），**Then** 系统计算并展示回测结果，包含收益率曲线图和关键指标
2. **Given** 用户查看回测结果，**When** 用户点击"交易明细"，**Then** 系统列出每次模拟交易的日期、价格、类型（买入/卖出）和盈亏
3. **Given** 回测期间5日均线上穿20日均线，**When** 该信号产生，**Then** 系统在回测记录中标记为"买入信号"
4. **Given** 回测期间5日均线下穿20日均线，**When** 该信号产生，**Then** 系统在回测记录中标记为"卖出信号"
5. **Given** 用户查看回测结果，**When** 用户修改参数（如均线周期从5/20改为10/30），**Then** 系统重新计算并展示更新后的回测结果
6. **Given** 模型框架已建立，**When** 未来添加新模型（如MACD、RSI等），**Then** 新模型可复用回测基础设施运行

---

### User Story 3 - 在线量化交易 (Priority: P3)

用户希望基于量化模型信号执行真实的股票交易。当双均线模型产生买入或卖出信号时，用户可选择手动确认执行或查看信号后自行决定。用户可下达买入/卖出指令，查看订单状态（已提交、已成交、已撤销），实时监控当前持仓的交易情况和浮动盈亏。

**Why this priority**: 在线交易是连接策略与收益的关键环节。需要先有数据（P1）和策略（P2）才能执行交易。交易涉及真实资金，因此相对靠后。

**Independent Test**: 连接模拟交易账户，通过模型信号或手动操作下达一笔交易指令，验证订单从提交到成交的完整流程。

**Acceptance Scenarios**:

1. **Given** 用户已绑定交易账户，**When** 双均线模型产生金叉买入信号，**Then** 系统通知用户并展示建议的买入价格和数量
2. **Given** 用户收到交易信号，**When** 用户确认执行买入，**Then** 系统向交易账户提交买入订单并显示订单状态
3. **Given** 用户有正在进行的订单，**When** 用户查看订单列表，**Then** 系统展示每笔订单的状态（待成交、已成交、已撤销）
4. **Given** 用户持有股票仓位，**When** 股票价格变动，**Then** 系统实时更新持仓的浮动盈亏
5. **Given** 用户希望手动交易，**When** 用户输入股票代码、价格和数量下达指令，**Then** 系统处理该交易指令

---

### User Story 4 - 账户持仓与收益监控 (Priority: P4)

用户希望随时查看交易账户的整体情况，包括当前持仓、可用资金、累计收益、当日盈亏等。系统以图表和列表形式直观展示账户资产变化趋势和持仓分布。

**Why this priority**: 持仓和收益监控是交易的配套功能，提供全局视图帮助用户评估策略效果和资金管理。

**Independent Test**: 在已有持仓数据的情况下，打开账户页面即可查看持仓明细和收益曲线，无需其他模块触发。

**Acceptance Scenarios**:

1. **Given** 用户已绑定交易账户，**When** 用户打开"账户概览"页面，**Then** 系统展示总资产、持仓市值、可用资金、累计收益率
2. **Given** 用户有多只持仓股票，**When** 用户查看持仓列表，**Then** 系统列出每只股票的持仓数量、成本价、现价、盈亏比例
3. **Given** 用户有历史交易记录，**When** 用户查看收益曲线，**Then** 系统以折线图展示账户总资产随时间的变化趋势
4. **Given** 当日有交易发生，**When** 用户查看当日统计，**Then** 系统展示当日盈亏金额和百分比

---

### Edge Cases

- 股票停牌或退市时，系统如何处理该股票的持仓展示和交易信号？
- 交易日非交易时段（如收盘后），系统是否仍生成交易信号？
- 网络中断导致实时行情数据延迟或缺失时，系统如何提示用户？
- 交易账户余额不足以执行模型建议的买入数量时，如何处理？
- 同一交易日同一股票多次触发均线交叉信号时，系统如何避免重复交易？
- 财务报表数据更新滞后（季报/年报发布延迟）时，如何标识数据时效性？
- 舆情数据源不可用时，舆情分析模块如何降级展示？

## Requirements *(mandatory)*

### Functional Requirements

**股票数据与研究**
- **FR-001**: 系统必须支持按股票代码搜索并展示股票的基本面信息（公司名称、所属行业、市值、市盈率、市净率等）
- **FR-002**: 系统必须展示股票近三年的关键财务报表数据（营业收入、净利润、ROE、资产负债率、每股收益等）
- **FR-003**: 系统必须收集并展示与股票相关的舆情资讯（新闻、公告、研报），并标注情感倾向（正面/负面/中性）
- **FR-004**: 系统必须展示股票所属行业板块信息及板块内相关股票的联动表现
- **FR-005**: 系统必须提供AI选股功能，基于基本面、技术面和舆情等多维度信息智能推荐股票

**量化模型与回测**
- **FR-006**: 系统必须支持可扩展的量化模型框架，允许持续添加新的量化模型
- **FR-007**: 系统必须实现双均线模型：当5日均线上穿20日均线时产生买入信号，5日均线下穿20日均线时产生卖出信号
- **FR-008**: 系统必须支持对量化模型进行历史回测，用户可设定回测时间范围和初始资金
- **FR-009**: 系统必须展示回测结果，包含总收益率、年化收益率、最大回撤、夏普比率、交易次数、胜率
- **FR-010**: 系统必须展示回测的详细交易明细（每次买卖的日期、价格、数量、盈亏）

**在线交易**
- **FR-011**: 系统必须支持用户绑定交易账户（通过券商接口）
- **FR-012**: 系统必须在模型产生交易信号时向用户推送通知
- **FR-013**: 系统必须支持用户手动确认或撤销模型建议的交易
- **FR-014**: 系统必须支持用户手动下达买入/卖出指令
- **FR-015**: 系统必须实时展示订单状态（已提交、部分成交、全部成交、已撤销）

**账户与监控**
- **FR-016**: 系统必须展示账户总览（总资产、持仓市值、可用资金、累计收益率）
- **FR-017**: 系统必须展示当前持仓明细（股票代码、名称、持仓数量、成本价、现价、盈亏比例）
- **FR-018**: 系统必须以图表形式展示账户资产变化趋势（资产曲线）和收益情况
- **FR-019**: 系统必须实时更新持仓的浮动盈亏

### Key Entities

- **股票 (Stock)**: 股票代码、名称、所属行业/板块、上市交易所
- **基本面数据 (Fundamental Data)**: 股票代码、市值、市盈率、市净率、每股收益、52周高低价等
- **财务报表 (Financial Report)**: 股票代码、报告期、营业收入、净利润、ROE、资产负债率、每股收益
- **舆情资讯 (Sentiment News)**: 股票代码、资讯标题、内容摘要、来源、发布时间、情感标签（正面/负面/中性）
- **量化模型 (Quant Model)**: 模型名称、模型类型、参数配置（如均线周期）、信号生成规则
- **交易信号 (Trade Signal)**: 股票代码、信号类型（买入/卖出）、触发模型、触发时间、建议价格
- **回测结果 (Backtest Result)**: 模型名称、股票代码、时间范围、初始资金、最终资金、总收益率、年化收益率、最大回撤、夏普比率、交易明细列表
- **订单 (Order)**: 股票代码、订单类型（买入/卖出）、价格、数量、状态、提交时间、成交时间
- **持仓 (Position)**: 股票代码、持有数量、成本价、现价、浮动盈亏
- **账户 (Account)**: 总资产、持仓市值、可用资金、累计收益率

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可在30秒内完成从输入股票代码到查看完整基本面信息及财务数据的操作
- **SC-002**: 90%以上的A股主板上市公司可查询到基本面、财务和舆情数据
- **SC-003**: 双均线模型回测在5秒内返回结果（含近一年历史数据的计算）
- **SC-004**: 回测结果与手动计算的收益率偏差不超过1%
- **SC-005**: 交易信号从触发到推送通知的延迟不超过30秒
- **SC-006**: 订单状态更新延迟不超过5秒
- **SC-007**: 账户持仓盈亏数据在行情更新后5秒内刷新
- **SC-008**: 系统支持同时监控至少20只股票的实时行情和交易信号
- **SC-009**: 用户可在单个界面内完成从策略信号查看到下单确认的完整交易流程

## Assumptions

- 目标用户为个人投资者，运行环境有稳定的互联网连接
- 数据范围为中国A股市场（沪深两市），暂不包含港股、美股等其他市场
- 实盘交易通过国金证券QMT量化交易平台接口执行
- 舆情数据来源于公开的财经新闻、公司公告及社交媒体讨论
- 首个版本仅包含双均线模型，模型框架设计支持后续扩展
- 系统面向单用户使用，不涉及多用户账户体系或权限管理
- 行情数据采用延迟不超过1分钟的非实时数据用于展示，实盘交易使用实时数据
- 交易信号的生成和通知均基于交易日内的交易时段
- 系统不提供投资建议，所有模型产生的信号仅供参考，用户自行决策
